# 이동형 지구 우주 항법 시스템 (Mobile Earth Navigation System)

## 프로젝트 전제
- "이동형 지구"가 미래에 제작된다는 가정 하의 우주 탐사 항법 시스템.
- 모든 설계 변천사는 /docs/DESIGN_HISTORY.md 에 기록되어 있다.

## 핵심 원칙 (요약)
- HYEAN = 최상위 비전(생존 지향 우주 지능 서비스), GWAN = 관측·해석·점수·판단·기억 엔진.
- Cloud Native(Docker/K8s/CI 등)는 정체성이 아니라 **수단**이다. 필요할 때만 도입한다.
- 모든 출력은 `known` / `estimated` / `simulated` / `hypothesis` / `fictional_future_scenario` 로 구분해 표기한다.
- 불확실한 데이터를 `confirmed`처럼 표현하지 않는다. (철학적 가설과 시스템 출력을 분리)
- GWAN v0.1은 **rule-based만** 구현한다. ML/딥러닝, 실제 센서 연동, 실제 3D UI는 보류.
- 상세 근거: 철학 → /docs/PHILOSOPHY.md · 확정 결정 → /docs/DECISIONS.md · 보류/다음 단계 → /docs/CONTEXT.md

## 작업 원칙
- 코드를 수정/추가하기 전, 반드시 DESIGN_HISTORY.md와 관련 모듈을 먼저 읽을 것.
- 기존 설계 결정과 충돌하는 변경은, 먼저 "기존 결정과 충돌함"을 명시하고 확인받을 것.
- 물리적 타당성(천체역학)과 SF적 설정을 코드 주석에서 구분 표기.
- 답변·커밋 메시지는 한국어로.

## 코드 작업 워크플로
- 변경 후: 포맷 → 테스트 실행 → 커밋 순서 준수.
- 커밋 메시지 형식: [모듈] 변경 요약 (이유)
- main에 직접 커밋 금지. 항상 브랜치 생성 후 PR 제안.

## 절대 금지
- 하드코딩된 비밀키/자격증명 작성 금지.
- 기존 핵심 항법 로직을 확인 없이 대규모로 갈아엎지 말 것.
