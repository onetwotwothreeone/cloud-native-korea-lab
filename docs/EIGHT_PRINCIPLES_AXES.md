## 1. 발상 — 인체는 작은 우주

동양의학은 인체를 작은 우주로 보고, **팔강변증(八綱辨證)** 의 네 축으로 몸의 상태를 읽었다. 같은 네 축으로 **우주 환경**을 읽어, HYEAN/GWAN의 예방 관측을 통합 네트워크로 확장하려는 구상이다. 이는 [PREVENTION_LAYER_SPEC.md](PREVENTION_LAYER_SPEC.md) 0절의 "동양의학적 통합 네트워크" 철학의 구체화다.

**중요(과설계 방지)**: 4축을 v0.1에 다 구현하면 H1 패턴(과설계, [DESIGN_HISTORY.md](DESIGN_HISTORY.md) ⚠️-4)을 반복한다. 그래서 지금은 **개념만 심고**, 예방 레이어는 확정된 **2축(severity×prevention)** 으로만 진행한다.

## 2. 4축 정의 (우주 관측으로의 대응)

| 팔강 축 | 인체 의미 | 우주 관측 대응 | 양극단의 위험 |
|---------|-----------|----------------|----------------|
| **허-실 (虛實)** | 부족 ↔ 과잉 | **에너지 신호 강도** | 허=보충 불가(에너지 고갈) / 실=과잉(ESD·플라스마 위험) |
| **한-열 (寒熱)** | 차가움 ↔ 뜨거움 | **열·복사 환경** (태양 거리, 복사열, 자외선) | 한=동결·에너지 부족 / 열=과열·복사 손상 |
| **표-리 (表裏)** | 겉(급성) ↔ 속(만성) | **위협의 급변 vs 누적** | 표=SPE 급성 피폭 / 리=GCR·내부 대전(charging) 만성 |
| **음-양 (陰陽)** | 안정·보호 ↔ 항진·노출 | **차폐·안정 vs 노출·교란** (자기장/플라스마 보호막 상태) | 음=과차폐·둔감 / 양=노출·교란 |

각 축은 "좋다/나쁘다"의 1차원이 아니라 **균형점에서 양극단으로의 이탈**을 본다([PHILOSOPHY.md](PHILOSOPHY.md) 1.6 균형 철학과 동형).

## 3. 현재 예방 레이어와의 대응 (이미 부분적으로 존재)

v0.1에서 새로 만든 2축이 4축의 일부에 **이미 부분 대응**한다. 즉 4축은 갑작스러운 신설이 아니라 기존 설계의 자연스러운 확장이다.

| 4축 | 현재 코드/설계의 대응 | 대응 상태 |
|-----|------------------------|-----------|
| **표-리 (급변↔누적)** | 예방 레이어의 **severity(순간) × prevention(흐름)** 2축 — 표=severity(급성), 리=prevention_status(누적 흐름) | 🟢 **부분 대응(구현됨)** |
| **허-실 (에너지)** | **`PreventionReading.energy_signal`(씨앗 필드로 실재)** + `energy_level`(잔량) + 예방의 `energy_consumption_rate` | 🟡 **자리 마련됨**(필드 존재, v0.1 계산 미사용 — [PREVENTION_LAYER_SPEC.md](PREVENTION_LAYER_SPEC.md) 5.1) |
| **한-열 (복사/열)** | 없음 (위험 신호에 일부 암묵 포함) | 🔴 미정의 |
| **음-양 (차폐/노출)** | 없음 (`risk_exposure`가 '노출'의 일부 그림자) | 🔴 미정의 |

→ **표-리·허-실은 이미 발을 들였고**, 한-열·음-양은 아직 측정 정의가 없다.

---

## 4. 범위와 미루기 (명시적 보류)

- **v0.1 예방 레이어 코드에 4축을 넣지 않는다.** 예방 레이어는 [PREVENTION_LAYER_SPEC.md](PREVENTION_LAYER_SPEC.md)의 2축(severity×prevention)으로만 구현한다.
- **표-리/음-양(및 한-열)의 '실제 측정 데이터 정의'** 는 **Space Data Source Map([CONTEXT.md](CONTEXT.md) 4.3) 단계, 즉 H4 이후로 미룬다.** 어떤 공개/시뮬레이션 데이터로 각 축을 측정할지는 그때 정의한다.
- 이 노트는 그 작업을 위한 **방향 표지(씨앗)** 일 뿐, 계약·코드가 아니다.

---

## 5. 다음 단계 (플랜에만 — 지금 구현하지 않음)

순서는 데이터가 먼저, UI는 나중이다.

1. (H4 이후) Space Data Source Map에서 **한-열·음-양·허-실의 측정 데이터 출처**를 정의.
2. 4축을 예방 레이어의 ruleset에 **규칙으로** 편입(엔진 무수정, [PREVENTION_LAYER_SPEC.md](PREVENTION_LAYER_SPEC.md) 2절의 정의/엔진 분리 구조 활용).
3. **4축 UI 시각화(레이더 차트 등)** 를 붙인다 — 실사용자에게 4축 균형을 한눈에 보여주는 좋은 방향. **단, Operator Interface UI는 현재 보류**([CONTEXT.md](CONTEXT.md) 4.12)이므로 이 단계는 **계획으로만** 기록한다. 데이터가 자리잡은 뒤 착수.

> 요약: **개념은 지금 심고, 데이터는 H4 이후, UI는 그다음.** v0.1은 2축만.
