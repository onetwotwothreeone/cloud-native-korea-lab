# 시각화 비전 (UI Vision) — 회장 선언, 씨앗 단계

> 상태: **씨앗(비전) 문서. 구현하지 않는다.** Operator Interface UI는 여전히 보류([CONTEXT.md](CONTEXT.md) 4.12)이며, 구현 착수는 **코어 성숙 후**(H3 판단 이중화 해소, 엔드포인트 노출 등)로 미룬다.
> 근거/연결: [PHILOSOPHY.md](PHILOSOPHY.md) 1.4.1(최종 목표 확장 선언) · [DECISIONS.md](DECISIONS.md) 2.11~2.12(Operator Interface 방향·레이아웃) · [EIGHT_PRINCIPLES_AXES.md](EIGHT_PRINCIPLES_AXES.md) 5절(4축 시각화 플랜) · [CONTEXT.md](CONTEXT.md) 3.8(시야 스케일).
> 분류 표기([PHILOSOPHY.md](PHILOSOPHY.md) 1.7): 본 문서는 미래 시각화의 방향 선언으로, 페이즈 1은 설계 구상, 페이즈 2의 "인공 대기권"은 이동형 지구 시대를 전제한 `fictional_future_scenario`에 속한다. 비전과 시스템 출력을 혼동하지 않는다.

---

## 0. 한 줄 요지

> HYEAN의 화면은 "우주를 색으로 읽는 구형 지도"다. 관측·분석 결과를 색상 구름으로 보여주고, 선택하면 근거를 설명하고, 위험과 기회는 스스로 알린다. 처음 보는 사람도 매뉴얼만 보면 이해할 수 있어야 한다.

---

## 1. 페이즈 1 — 구형 네비게이터 (Spherical Navigator)

회장 비전(원문 뜻 그대로 보존):

- **반지름 약 1 AU의 구형 지도.** 반경(줌)은 지도 앱처럼 자유롭게 조정 가능하다.
  - 연결: "약 1 AU"는 고정값이 아니라 기본 시야이며, [CONTEXT.md](CONTEXT.md) 3.8의 가변 스케일(Local/Tactical/Regional/Strategic)과 정합. 계약에는 이미 `RangeScale` enum이 존재한다.
- **가능한 많은 분석을 '색상 구름(color cloud)'으로 시각화.** 분광·마이크로파·플라즈마·원자 분석 등, GWAN이 수행할 수 있는 분석 결과를 공간 위에 색의 구름으로 겹쳐 보여준다.
- **색상 원칙**:
  - 안전 = **초록**, 에너지원 = **보라**.
  - 안전 관련 색상은 **국제 관례(ISO 안전색: 초록=안전, 빨강=위험, 노랑=주의)** 를 따른다.
  - 국제 표준이 없는 영역(에너지원 등)은 **자체 팔레트를 정의**해 일관되게 사용한다.
  - 연결: "색상 의미를 애매하게 만들지 않는다"는 [DECISIONS.md](DECISIONS.md) 2.12의 비목표와 동일 원칙. 불확실한 데이터를 확정된 것처럼 색으로 단정하지 않는다(1.7 분류 원칙의 시각화 버전).
- **포인트 선택 → 사이드바 상세.** 지도에서 포인트를 고르면 사이드바에 거리·점수·분광 해석·출처·불확실성·추천 행동 등 상세 정보를 보여준다.
- **자동 알림.** 매우 위험한 포인트, 또는 HYEAN의 목표(생존·발견·에너지 — [PHILOSOPHY.md](PHILOSOPHY.md) 1.4.1)에 부합하는 포인트를 발견하면 시스템이 스스로 알린다.
- **접근성 원칙**: 처음 보는 사람도 **매뉴얼만 보면 이해할 수 있는** 화면이어야 한다. 전문가 전용 암호 같은 UI를 만들지 않는다.

## 2. 페이즈 2 — 인공 대기권(방어막) 모니터링

회장 비전(원문 뜻 그대로 보존, `fictional_future_scenario` 전제):

- 이동형 지구 시대의 **인공 대기권(방어막) 상태를 시각적으로 표시**한다. 안정적인지 아닌지 **한눈에** 들어와야 한다.
- **전문가용 상세 상태는 사이드바로.** 정확하고 판단에 도움이 되는 정보를 깊이 있게 제공한다(한눈 요약과 전문 상세의 이중 구조 — 페이즈 1과 동일한 문법).
- **범용 모니터링**: 이 구조는 이동형 지구만이 아니라 **기존 지구, 그리고 미래에 인류가 생존하게 될 다른 행성**의 대기·방어막 모니터링에도 적용 가능하다. → [PHILOSOPHY.md](PHILOSOPHY.md) 1.4.1의 **수혜자 3중**(이동형 지구 내 인류 / 기존 지구 인류 / 미래 타 행성 인류)과 직접 연결. UI 차원에서도 HYEAN은 한 척의 시스템이 아니라 인류 공용 인프라다.

## 3. 중요한 연결 — 이 비전은 기존 Data Contract의 '목적지'다

이 비전은 신규 발명이 아니다. **기존 계약 5 package 구조가 이미 이 화면을 위해 설계되어 있다.** 즉 UI 비전은 계약 구조의 목적지(destination)이며, 계약은 그 비전의 데이터 공급선이다.

| 비전 요소 | 기존 Data Contract 대응 |
|-----------|--------------------------|
| 구형 지도 + 색상 구름 + 포인트 | `spatial_visualization_package` (`RangeScale`, `Position3D`, `MarkerStyle.color_group`) |
| 포인트 선택 → 상세 | `sidebar_intelligence_package` |
| 위험·기회 자동 알림 | `alert_feed_package` (severity·priority 정렬 지원 이미 구현) |
| 판단 근거·불확실성 표시 | `decision_report_package` / `uncertainty_package` |
| (H4 이후) 흐름·예방 상태 표시 | `payload.prevention` (`PreventionReport` — severity×prevention 두 축) |

색상 구름의 "무엇을 색으로 칠할 것인가"는 팔강변증 4축([EIGHT_PRINCIPLES_AXES.md](EIGHT_PRINCIPLES_AXES.md)) 및 에너지 축(`energy_signal` 씨앗)과 만난다 — 다만 그 측정 정의는 Space Data Source Map 단계의 일이다.

---

## 4. 범위와 순서 (명시적 보류)

- **지금 구현하지 않는다.** 프론트엔드 프레임워크·3D 라이브러리·색상 팔레트 확정 등은 전부 미정([CONTEXT.md](CONTEXT.md) 3.7)이며 이 문서도 그것을 확정하지 않는다.
- 순서는 **코어 → 데이터 → UI**: ① H3(판단 이중화 해소)·엔드포인트 노출 등 코어 성숙 → ② Space Data Source Map으로 측정 데이터 정의 → ③ 그 데이터 위에 본 비전의 화면을 얹는다.
- 이 문서의 역할은 방향 표지다: **계약·코어를 다듬을 때 "이 화면으로 간다"는 목적지를 잊지 않기 위함.**
