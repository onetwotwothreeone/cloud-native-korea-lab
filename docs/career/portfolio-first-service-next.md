# Portfolio First, Service Next

## Strategy

Cloud Native Korea Lab follows this sequence:

```text
Portfolio first
  -> internal tool
  -> public demo
  -> closed beta
  -> paid pilot
  -> real service
```

The project should first become a strong xAI portfolio. After that, it should become a real service.

## Why Portfolio First

A portfolio proves technical ability.
A service proves market value.

If the project starts as a service too early, the scope may become too large. If it ends as only a portfolio, the project may waste its long-term potential.

The best path is:

> Build a small but complete portfolio first, then extend it into a real service.

## Phase 1. Portfolio MVP

### Goal

Create a project that can be shown to xAI recruiters and engineers.

### Requirements

- clear problem definition
- FastAPI API
- PostgreSQL data model
- Docker Compose execution
- Kubernetes manifests
- GitHub Actions test
- README documentation
- sample evaluation data

## Phase 2. Portfolio Polish

### Goal

Make the project easy to understand in 3 minutes.

### Requirements

- English README
- architecture diagram
- problem-definition document
- API examples
- technical trade-offs
- known limitations
- next-step roadmap

## Phase 3. Internal Tool

### Goal

Use the platform personally to evaluate Korean AI outputs and learning materials.

### Requirements

- register samples
- evaluate samples
- mark ambiguous cases
- save judgments
- export CSV / JSON

## Phase 4. Public Demo

### Goal

Allow other people to try the service with safe demo data.

### Requirements

- simple web UI
- demo sample data
- no private user upload at first
- result summary page
- project intro page

## Phase 5. Closed Beta

### Goal

Let a small group of users test the workflow.

### Initial Users

- AI developers
- Korean language educators
- Cloud Native Korea Lab community members
- people building Korean AI products

## Phase 6. Paid Pilot

### Goal

Validate whether users will pay for Korean AI evaluation support.

### Possible Paid Offers

- Korean AI answer quality report
- Korean dataset quality check
- Korean voice sample quality review
- AI tutor response evaluation
- custom evaluation rubric design

## Phase 7. Real Service

### Goal

Operate a real Korean AI data quality and evaluation platform.

### Required Later

- authentication
- data security
- privacy policy
- terms of service
- admin page
- user data deletion flow
- monitoring
- backup
- incident response

## Non-negotiable Rule

If the user later works with xAI or any other AI company, internal data, confidential tasks, and private work materials must never be used in this public service.

Only public data, synthetic data, self-created samples, or user-consented data can be used.

## Final Message

> Prove skill with the portfolio. Prove value with the service.
