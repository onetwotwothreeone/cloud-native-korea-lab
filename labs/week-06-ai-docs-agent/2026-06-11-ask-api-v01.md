# Cloud Native AI Docs Agent v0.1 - /ask API Lab

## 1. 실습 제목

Cloud Native AI Docs Agent v0.1의 /ask API 만들기

## 2. 실습 목표

기존 Mini Platform 앱에 POST /ask API를 추가해서, 사용자가 클라우드 네이티브 질문을 보내면 정해진 답변 템플릿으로 응답하게 만든다.

## 3. 왜 /ask API가 필요한가?

/ask API는 공식 문서 기반 AI Docs Agent의 질문 접수 창구다.

사용자는 /ask API로 질문을 보내고, 서버는 질문을 받아 정해진 답변 구조로 응답한다.

이번 v0.1에서는 실제 AI 모델이나 RAG를 붙이지 않고, 고정된 예시 응답으로 API 구조만 만든다.

## 4. 입력 JSON

```json
{
  "question": "Kubernetes에서 Service는 왜 필요한가요?"
}
```
## 5. 출력 JSON 구조

```
{
  "question": "사용자 질문",
  "answer": {
    "summary": "핵심 요약",
    "easyExplanation": "초보자용 쉬운 설명",
    "keyTerms": [],
    "practice": {
      "title": "실습 제목",
      "commands": []
    },
    "note": "현재는 고정 응답"
  },
  "sources": [],
  "version": "v0.1.0"
}

```
## 6. 실행한 명령어

```
cd ~/cloud-native-korea-lab/mini-platform/app
npm install
npm start

```
새 터미널에서 테스트:

```
curl -X POST http://localhost:3000/ask \
  -H "Content-Type: application/json" \
  -d '{"question":"Kubernetes에서 Service는 왜 필요한가요?"}' | python3 -m json.tool
```
## 7. 성공 확인 결과
POST /ask 요청 성공
사용자가 보낸 question 값이 응답에 포함됨
answer.summary 확인
answer.easyExplanation 확인
keyTerms 확인
practice.commands 확인
sources 확인
version 확인
## 8. 오늘 배운 점
/ask API는 AI Docs Agent의 질문 접수 창구다.
app.use(express.json())이 있어야 JSON 요청 본문을 읽을 수 있다.
실제 AI와 RAG를 붙이기 전에도 API 입력과 출력 구조를 먼저 설계할 수 있다.
